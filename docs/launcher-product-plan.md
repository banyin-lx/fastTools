# 快捷启动软件产品规划草案

## 1. 项目目标

打造一款类似 `uTools` 与 `Flow Launcher` 的桌面快捷启动软件，基于 `C# + WPF` 开发，重点提供以下能力：

- 全局呼出，快速搜索与启动
- 命令执行与工作流触发
- 文件、应用、网页、系统设置的一体化检索
- 可扩展的插件体系
- 现代化、简洁、动效自然的桌面 UI

这份文档是一个“可裁剪的功能池 + 周期草案”，可在后续评审时按优先级删减。

---

## 2. 产品定位

### 2.1 核心定位

面向高频电脑用户的效率启动器，帮助用户通过键盘完成：

- 启动应用
- 搜索文件
- 执行系统命令
- 打开网页或搜索引擎
- 调用插件功能
- 触发自动化动作

### 2.2 目标用户

- 开发者
- 设计师
- 办公重度用户
- 运维 / 测试 / 产品经理
- 喜欢键盘流操作的 Windows 用户

### 2.3 产品关键词

- 快
- 稳
- 美观
- 可扩展
- 低打扰
- 可配置

---

## 3. 设计方向

### 3.1 UI 风格建议

- 整体风格偏现代桌面工具，而不是传统 Win32 工具感
- 使用圆角、半透明、阴影、层次感卡片式布局
- 搜索框作为视觉核心，结果列表突出层级与状态
- 支持浅色 / 深色主题
- 动效克制，重点放在呼出、聚焦、结果切换、面板展开
- 字体、间距、图标需统一，避免“插件拼装感”

### 3.2 交互原则

- `Alt + Space` 或自定义快捷键全局呼出
- 键盘优先，鼠标辅助
- 常用操作尽量在 1 到 2 次按键内完成
- 搜索结果实时反馈，响应尽量小于 100ms
- 所有高频功能都应可配置别名 / 关键词 / 排序权重

## 4. 功能范围总览

## 4.1 MVP 必做功能

### A. 全局启动与搜索

- 全局快捷键呼出主窗口
- 输入关键字后实时搜索
- 搜索结果支持键盘上下选择、回车执行、Esc 关闭
- 结果分组展示：应用、文件、命令、网页、插件
- 最近使用记录与权重排序
- 拼音首字母 / 模糊匹配 / 别名匹配

### B. 应用启动

- 索引系统开始菜单应用
- 索引用户自定义应用目录
- 启动 exe、lnk、uwp 应用
- 支持“管理员方式运行”
- 支持自定义关键词与别名

### C. 文件与文件夹检索

- 配置需要索引的目录
- 按文件名检索文件 / 文件夹
- 常见文件操作快捷动作：
  - 打开
  - 打开所在目录
  - 复制路径
  - 以默认程序打开

### D. Web 与浏览器动作

- 输入网址直接打开
- 使用默认搜索引擎搜索关键字
- 支持自定义搜索源：
  - 百度
  - Google
  - Bing
  - GitHub
  - Bilibili

### E. 系统命令与快捷动作

- 锁屏
- 重启
- 关机
- 睡眠
- 打开任务管理器
- 打开控制面板 / 设置页
- 清空回收站
- 执行自定义命令或脚本

### F. 现代化主界面

- 中央悬浮式启动面板
- 顶部搜索输入区
- 中部结果列表区
- 底部操作提示区
- 支持主题切换
- 支持窗口显示动画与失焦自动隐藏

### G. 设置中心

- 全局快捷键设置
- 索引目录设置
- 搜索排序设置
- 主题设置
- 插件开关
- 自定义命令管理

## 4.2 第二阶段增强功能

### A. 插件系统

- 定义插件接口规范
- 支持本地插件加载
- 插件可返回搜索结果项
- 插件可提供独立设置页
- 插件市场能力预留

### B. 剪贴板增强

- 剪贴板历史记录
- 文本 / 图片分类展示
- 快速复制回填
- 置顶常用条目
- 清空历史

### C. 快捷短语 / 文本片段

- 常用回复模板
- 代码片段
- 邮箱 / 地址 / 电话等固定文本
- 通过关键字快速插入

### D. 计算与转换

- 基础四则运算
- 汇率 / 单位 / 时间转换
- 日期差计算
- 常用公式快捷计算

### E. 窗口与系统增强

- 窗口置顶
- 窗口透明度调整
- 快速切换音量 / 亮度
- 多显示器快捷操作

## 4.3 第三阶段扩展功能

### A. 工作流与自动化

- 串联多个动作形成 workflow
- 条件分支与参数输入
- 调用本地脚本、PowerShell、命令行工具
- 支持导入 / 导出工作流

### B. AI 能力接入

- 自然语言触发动作
- 文本摘要 / 翻译 / 改写
- 代码片段解释
- AI 插件统一接入层

### C. 云同步

- 配置同步
- 插件同步
- 快捷短语同步
- 用户偏好同步

### D. 插件市场

- 插件安装、升级、卸载
- 插件评分与推荐
- 版本兼容校验

## 6. 关键技术模块建议

- `Shell / App Host`：程序生命周期、单实例、托盘、全局热键
- `Search Core`：索引、召回、模糊匹配、排序
- `Launcher Providers`：应用、文件、命令、网页、系统项的数据提供器
- `Action Engine`：执行动作、参数传递、管理员权限处理
- `Plugin SDK`：插件接口、插件生命周期、隔离与加载
- `Settings & Persistence`：配置、历史记录、缓存、日志
- `UI Layer (WPF)`：主窗口、结果项模板、主题、动画、设置页

## 7. 非功能需求

### 性能

- 冷启动尽量控制在 2 秒内
- 热唤起尽量控制在 300ms 内
- 普通搜索结果响应尽量控制在 100ms 内

### 稳定性

- 插件异常不应导致主程序崩溃
- 索引更新应支持后台运行
- 配置损坏时应可自动回退默认配置

### 可扩展性

- 搜索源使用统一接口抽象
- 动作执行器与展示层解耦
- 插件能力预留版本管理

### 可维护性

- 模块边界清晰
- 日志与异常追踪完整
- 预留单元测试与集成测试

## 9. 迭代建议

### 9.1 第一轮优先做什么

- 先把“呼出快、搜索快、执行快”做扎实
- 先不急着做复杂插件市场
- UI 先完成统一视觉语言，再扩功能

### 9.2 第二轮再补什么

- 插件生态
- 工作流
- 剪贴板与短语管理深度功能
- AI 入口

### 9.3 不建议一开始就做的内容

- 云同步
- 在线插件市场
- 复杂账号系统
- 过重的脚本沙箱

---

## 10. 风险点

- 全局热键、窗口焦点、输入法兼容性处理较容易出问题
- 文件索引范围过大时，性能与内存占用可能上升
- 插件系统若缺少隔离，容易带来稳定性问题
- WPF 想做出“现代化高级质感”，需要较强的样式系统设计
- 权限相关动作可能涉及 UAC 与执行策略问题

---

## 11. 建议的首版范围结论

如果目标是尽快做出一个能用、能展示、能继续扩展的版本，建议首版只保留：

- 全局呼出
- 应用启动
- 文件检索
- 网页搜索
- 系统快捷动作
- 基础设置中心
- 深浅色主题
- 统一现代化主界面

在这个基础上，第二阶段优先加入：

- 插件系统
- 剪贴板历史
- 快捷短语
